# vs-token-safer 1.0 — 주요 기능 정리

한 줄: **에이전트가 저장소를 읽는 대신 말을 거는 레이어.** 모든 코드 검색을 공식 언어 서버 인덱스로 강제 통과시키고 `file:line`으로 자른다. 로컬 전용, 본문 없음.

## 정밀도 사다리 (제품)

| 발판 | 언제 | 엔진 | 보장 |
| --- | --- | --- | --- |
| **EXACT** | 이름 알고 툴체인 있음 | clangd · Roslyn · tsserver · pyright | 의미상 확정 (참조는 참조, 리네임 안전) |
| **SYNTACTIC** | 툴체인 없음 | tree-sitter (문법 36개, 17개 손봄) | 진짜 선언, 설정 제로, 공유용 `.vts-index/symbols.jsonl` |
| **FUZZY** | 의도만 앎 | 저장소 자체 이름+주석에서 캔 개념 사전 (임베딩 없음) | 결정적, 들여다볼 수 있음, RM3 + 임포트 그래프 + 깃 동시변경 |
| **SECTION** | 코드 아닌 문서/설정 | tree-sitter (md/html/css/toml/yaml) | 파일 전체 아니라 헤딩/셀렉터 하나 |

발판은 **이어진다.** FUZZY가 이름 건짐 → EXACT로 올라가 확정. EXACT 빗나감 → FUZZY로 떨어짐. 모든 답은 발판 이름 + 잘림/타임아웃 여부를 적은 **완전성 증명서**를 단다.

## 다섯 가지 메커니즘

1. **강제 훅 (초안 이후 신규, 1.0 헤드라인).** 네 표면:
   - 소스 뒤지는 Bash grep/rg/`find -name` → **그 자리에서** `vts` 쿼리로 다시 씀 (식별자→`search_symbol`, 리터럴→`search_text`, `find -name`→`find_files`).
   - 읽기 전용 본문 덤프 (`sed -n`/`cat`/`head`/`tail`/`awk`, 인터프리터 `-c`/`-e` 원라이너) → **`read_symbol`로 유도** (편집 전 파일 전체 Read를 죽임).
   - Grep/Glob 도구 심볼 사냥이나 코드 파일 글롭 → 바로 쓸 호출과 함께 **막음**.
   - Edit/MultiEdit가 선언 통째로 교체 → `replace_symbol_body`/`insert_symbol`로 **넛지**, 반복 무시 때 격상.
   - 노브: `VTS_REWRITE=0`, `VTS_GREP_BLOCK=0`, `VTS_ENFORCE=0`, `VTS_EDIT_WARN`/`VTS_EDIT_BLOCK_AFTER`. i18n EN/KO.
2. **토큰 상한 코어.** LSP 결과 → `kind name @ file:line`, 상한, `… N more`. 참조는 공유 디렉터리 접두어를 한 번 빼고 파일당 한 줄로 접힘. 잘린 결과는 전체 집합을 복구 파일로 티.
3. **심볼 단위 편집.** `replace_symbol_body`/`insert_*`/`safe_delete`가 아웃라인으로 이름 → 선언 해석, 정확한 스팬에 텍스트 이어붙임. 기본 미리보기, `apply=true`로 씀, `safe_delete`는 참조 남아 있으면 거부. 파일 전체 Read 없음.
4. **헤드리스 LSP 클라이언트.** 공식 엔진을 stdio로 소유. 프로젝트 루트를 **호출마다** 해석해 서버 하나가 세션이 건드리는 모든 저장소에 답함. 풀링 + 상한 (`VTS_MAX_BACKENDS` + 유휴 리퍼).
5. **savings + discover.** 검색별 절감 토큰 로컬 원장 (`vts savings`, 30일 그래프). `vts discover`는 최근 세션에서 인덱스를 **우회한** 검색을 훑음. 성공만 아니라 포착률까지.

## 큰 저장소 경화 (1.0이 실제 UE 트리에서 살아남는 이유)

- 크롤 위험 트리 (clangd 빨리 못 답함) → `read_symbol`/`find_references`/`document_symbols`/심볼 편집이 멈추는 대신 **커밋된 tree-sitter 인덱스로 선점**.
- 셀프서브 커밋 인덱스 경로: 인덱스 없는 트리도 빠르게 답 + `vts_index` 빌드 힌트.
- 언리얼 `Content/` (바이너리 애셋 74만 7천) 건너뜀 → uasset/ini 검색 안 멈춤.
- 콜드 첫 답 369초 → 51초 (하위 트리 인덱스 + 정적 파일 로드). 7.8배.
- 강제는 충실한 답 못 내면 **다시 쓰기를 되돌림**. 틀린 답 안 내보냄.

## 수치

- grep 대비 **토큰 87% 절감** (3언어 150파일 벤치, 47,547 → 6,195).
- 설정 제로 tree-sitter 발판 **~99%**.
- 실제 언리얼 엔진 5 트리 **~138배** (한 탐색 28만 2천 → ~2천 토큰).

## 세 약속 (절대 양보 없음)

1. **로컬 전용, 전송 제로** — 모든 엔진 stdio/localhost, 기계 밖 나가는 것 없음.
2. **정확성은 공식 엔진** — clangd/Roslyn/tsserver/pyright, 재구현 없음, 남의 인덱스 소스 위에 안 씌움.
3. **`file:line` 상한, 본문 없음, 정확성에 임베딩 없음** — 근사는 FUZZY 발판에만 살고, 표시됨.

## 정직한 모서리 (1.0로 안고 감)

- 퍼지는 임베딩 폴백 없음 — 저장소에 어휘적 다리 없는 순수 유의어는 빗나감 (재현율보다 결정성 택함).
- 강제는 **안전한 곳 막고 나머지 경고**. 심볼 편집은 경고 + 격상, 하드 브레이크 없음.
- 심볼 편집 채택률은 **측정됨(~2%), 해결 안 됨** — 읽기 쪽 개입 출시, 보답은 아직 쌓이는 실제 세션 숫자.
- tree-sitter 0.26 ABI 범프 세워둠, `typescript` `^6` 고정 (TS7 네이티브 포트가 `tsserver.js` 뺌).

**동반:** `gamedev-log-analyzer` (거대 에디터/빌드 로그에 같은 로컬 전용 규율).
