# vs-token-safer 1.0 — major feature summary

One-line: **the layer an agent talks to instead of reading the repository** — every code search forced through the official language-server index and capped to `file:line`, local-only, no bodies.

## The precision ladder (the product)

| Rung | When | Engine | Guarantee |
| --- | --- | --- | --- |
| **EXACT** | name known + toolchain present | clangd · Roslyn · tsserver · pyright | semantic ground truth (a ref is a ref; rename is safe) |
| **SYNTACTIC** | no toolchain | tree-sitter (36 grammars, ~17 tuned) | real declarations, zero setup; shareable `.vts-index/symbols.jsonl` |
| **FUZZY** | you only know intent | concept dictionary from the repo's own names+comments (no embeddings) | deterministic, inspectable; RM3 + import-graph + git co-change |
| **SECTION** | doc/config, not code | tree-sitter (md/html/css/toml/yaml) | address one heading/selector, not the whole file |

Rungs **connect**: FUZZY surfaces a name → climbs to EXACT to confirm; EXACT miss → drops to FUZZY. Every answer carries a **completeness certificate** naming its rung + whether it was capped/timed-out.

## The five mechanisms

1. **Enforcement hook (new-since-draft, the 1.0 headline).** Four surfaces:
   - Bash grep/rg/`find -name` over source → **rewritten in place** to the `vts` query (identifier→`search_symbol`, literal→`search_text`, `find -name`→`find_files`).
   - Read-only body dumps (`sed -n`/`cat`/`head`/`tail`/`awk`, interpreter `-c`/`-e` one-liners) → **steered to `read_symbol`** (kills the pre-edit whole-file Read).
   - Grep/Glob tool symbol-hunt or code-file glob → **blocked** with a ready call.
   - Edit/MultiEdit replacing a whole decl → **nudged** to `replace_symbol_body`/`insert_symbol`, escalating on repeated ignores.
   - Knobs: `VTS_REWRITE=0`, `VTS_GREP_BLOCK=0`, `VTS_ENFORCE=0`, `VTS_EDIT_WARN`/`VTS_EDIT_BLOCK_AFTER`. i18n EN/KO.
2. **Token-capping core.** LSP result → `kind name @ file:line`, capped, `… N more`. Refs collapse to one row/file with shared dir prefix factored out. Truncated results tee full set to a recovery file.
3. **Symbol-level editing.** `replace_symbol_body`/`insert_*`/`safe_delete` resolve a decl by name via the outline, splice at its exact span. Preview by default, `apply=true` writes, `safe_delete` refuses while referenced. No whole-file Read.
4. **Headless LSP client.** Owns the official engine over stdio; project root resolved **per call** so one server answers every repo a session touches. Pooled + bounded (`VTS_MAX_BACKENDS` + idle reaper).
5. **Savings + discover.** Local ledger of tokens-saved (`vts savings`, 30-day graph). `vts discover` scans recent sessions for searches that **bypassed** the index — catch-rate, not just wins.

## Big-repo hardening (why 1.0 survives a real UE tree)

- Crawl-risk trees (clangd can't answer fast) → `read_symbol`/`find_references`/`document_symbols`/symbol-edit **pre-empt to the committed tree-sitter index** instead of hanging.
- Self-serve committed-index path: no-index tree answers fast + gets a `vts_index` build hint.
- Skips Unreal `Content/` (747k binary assets) so uasset/ini searches don't hang.
- Cold first-answer 369s → 51s (subtree index + static-file load). 7.8×.
- Enforcement **reverts a rewrite** when it can't produce a faithful answer rather than ship a wrong one.

## Numbers

- **87% fewer tokens than grep** on a 3-language 150-file benchmark (47,547 → 6,195).
- **~99%** on the zero-setup tree-sitter tier.
- **~138×** on a real Unreal Engine 5 tree (282k → ~2k tokens on one hunt).

## The three promises (never traded)

1. **Local-only, zero transmission** — every engine over stdio/localhost; nothing leaves the machine.
2. **Official engines for correctness** — clangd/Roslyn/tsserver/pyright; never reimplemented, never a third-party index over your source.
3. **Capped `file:line`, no bodies, no embeddings for correctness** — approximation lives only on the FUZZY rung, and it's labeled.

## Honest edges (carried into 1.0)

- Fuzzy has no embedding fallback — a pure synonym with no lexical bridge misses (deterministic-over-recall by choice).
- Enforcement is **block-where-safe, warn-elsewhere**; symbol-edit is warn-only + escalate, never a hard break.
- Symbol-edit adoption is **measured (~2%), not solved** — read-side intervention shipped; the payoff is a real-session number still accruing.
- tree-sitter 0.26 ABI bump parked; `typescript` pinned `^6` (TS7 native port drops `tsserver.js`).

**Companion:** `gamedev-log-analyzer` (same local-only discipline for giant editor/build logs).
